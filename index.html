PROBLEMA DE LA SUMA DE SUBCONJUNTOS

DEFINICIÓN


En esta ocasión vamos a hablar de un problema que es muy sencillo de entender, pero que resulta muy complicado resolver. Lo normal en informática, vamos. Se trata del problema de la suma de los subconjuntos: Dado un conjunto A = {a1, a2,..., an} de 'n' números no negativos y otro número positivo 'S', se trata de encontrar un subconjunto de 'A' cuya suma es 'S'. Se supone que el conjunto de entrada esa único, es decir no se presentan duplicados.

De manera similar lo podemos expresar como: Dado un conjunto de enteros y un entero 'S', ¿existe algún subconjunto cuya suma sea 'S'
El problema de la suma de subconjuntos es un problema importante en la teoría de la complejidad y en la criptografía. Además, se trata de un problema NP-Completo, es decir, es muy fácil verificar si una solución es correcta (tan sólo hay que sumar los valores y ver si da 'S'), pero al día de hoy no se conoce ningún algoritmo que lo resuelva de manera eficiente (en un tiempo que sea del orden de un polinomio).

Ejemplo: 

        Input:
            A = { 7, 3, 2, 5, 8 }
            S = 14
        Output:
            True, Existe un subconjunto que completa la suma
            El subconjunto { 7, 2, 5 } suma 14

ALGORITMOS DE SOLUCIÓN

> SOLUCIÓN POR FUERZA BRUTA (RECURSIÓN)

El algoritmo de fuerza bruta verifica todos los posibles subconjuntos de 'A' y, para cada uno de ellos, compararía la suma al total buscado.
Para implementar el algoritmo de fuerza bruta utilizando recursión, consideramos los siguientes casos:
 - Incluir el elemento actual en el subconjunto y continuamos la recursión para los elementos restantes con el total restante.
 - Excluir el elemento actual del subconjunto y continuamos la recursión para los artículos restantes. ... [leer más]


De igual forma excluiremos cualquier elemento que sea mayor a la suma y continuaremos con el siguiente elemento del arreglo.
Finalmente regresamos TRUE si obtenemos un subconjunto que sume 'S' al incluir o excluir el elemento actual, se regresa FALSE en cualquier otro caso.
El caso base de la recursión se da cuando no quedan elementos en 'A' o la suma se vuelve negativa. Devuelve TRUE cuando la suma se convierte en 0, es decir, se encuentra el subconjunto.
La solución por fuerza bruta puede probar todos los subconjuntos del conjunto dado en el peor de los casos. Por lo tanto, la complejidad temporal de esta solución es exponencial.

Complejidad Temporal: O(2^n)
Complejidad Espacial: O(1)

> SOLUCIÓN TOP-DOWN

Este problema tiene una subestructura optima, eso significa que el problema se puede dividir en "subproblemas" más pequeños y simples, que se pueden dividir en subproblemas aún más simples y pequeños hasta que la solución se vuelva trivial.
Podemos notar que la solución por fuerza bruta tiene problemas superpuesto pues como se podrá observar en la animación, podemos ver que los mismos subproblemas se calculan repetidamente. ... [leer más]


Sabemos que los problemas con la subestructura óptima y los subproblemas superpuestos se pueden resolver mediante la programación dinámica, donde las soluciones de los subproblemas se memoizan en lugar de calcularse una y otra vez.
Es esto lo que realizamos en la solución Top-Down, donde primero dividimos el problema en subproblemas y luego calculamos y almacenamos los valores.
En este método, seguimos utilizando recursión, pero hacemos uso de un array de 2 dimensiones, de tamaño n x sum, y que inicializaremos con -1 o cualquier valor negativo. Con este método, evitaremos hacer llamadas recursivas que se repitan pues en esta matriz, iremos guardando los valores de las llamadas previas.
Los casos base de esta solución se dan cuando la suma es igual a 0 o cuando no existen más elementos en el arreglo, regresamos TRUE y FALSE, respetivamente.

La complejidad temporal de esta solución es O(n × sum) y se requiere de O(n × sum) espacio extra, donde n es el número de elementos y sum es la suma de todos los elementos de la entrada.

Complejidad Temporal: O(n × sum)
Complejidad Espacial: O(n × sum)

> SOLUCIÓN BOTTOM-UP

De igual forma podemos resolver este problema con el enfoque Bottom-Up, donde primero resolvemos los subproblemas más pequeños, luego resolvemos los subproblemas más grandes a partir de ellos.
Crearemos una matriz de tamaño (n+1) x (S +1), donde n es el numero de elementos y S la suma deseada.

La idea es la siguiente:
    if (A[i-1] > j)
        DP[i][j] = DP[i-1][j]
    else
        DP[i][j] = DP[i-1][j] OR DP[i-1][j-A[i-1]]

Esto significa que, si el elemento actual es mayor o igual al valor de la suma actual, copiaremos el valor de los casos anteriores.
Y si el valor de la suma actual es mayor que el elemento 'i-ésimo', veremos si alguno de los estados anteriores ya ha experimentado la suma = 'j', o cualquier estado anterior experimentó un valor 'j - A [i]' que resolverá nuestro propósito.

Esta solución tiene la misma complejidad que el enfoque Top-Down pero sin hacer uso de la recursión.

Complejidad Temporal: O(n × sum)
Complejidad Espacial: O(n × sum)
